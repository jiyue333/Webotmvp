# docs/

## 文件职责
- 维护项目文档索引与阅读顺序。
- 作为架构、接口、运行维护文档的统一入口。

## 边界
- 文档仅描述 MVP 范围与实现约定，不替代源码中的最终行为定义。
- 复杂设计变更以代码与迁移为准，文档负责同步说明。

## 文档索引
- `architecture.md`：系统结构、分层边界、关键数据流。
- `api.md`：MVP 路由语义、响应约定、错误处理约束。
- `runbook.md`：本地启动、排障、健康检查、回滚建议。
